## Database Setup

### 1. Клонирование репозитория
git clone <repo>
cd project

### 2. Установка зависимостей
npm install

### 3. Создание и миграция БД
npm run db:migrate

### 4. Заполнение начальными данными (только один раз!)
npm run db:seed

### 5. Запуск проекта
npm run dev

## Daily Development / Разработка (данные уже есть)

### Обычный запуск - seed НЕ нужен
npm run dev

### Проверить состояние БД
npm run db:check 

## Добавление новой миграции

### 1. Изменить schema.ts
### 2. Сгенерировать миграцию
npm run db:generate

### 3. Применить миграцию (данные сохранятся!)
npm run db:migrate

### 4. Продолжить работу
npm run dev

## Когда запускать seed

| Сценарий | Команда | Когда |
|----------|---------|-------|
| **Первая установка** | `npm run db:seed` | Один раз после клонирования |
| **Ежедневная разработка** | `npm run dev` | Каждый раз, seed НЕ нужен |
| **После git pull** | `npm run db:migrate` | Если есть новые миграции |
| **Reset для тестов** | `npm run db:reset:dev` | Когда нужны чистые данные |
| **Production** | Никогда! | Используйте миграции |
